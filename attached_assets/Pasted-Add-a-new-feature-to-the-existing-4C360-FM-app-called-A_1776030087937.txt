Add a new feature to the existing 4C360 FM app called:

“Asset Expert Voice Copilot” for the PPM page

Important:
- Do NOT rebuild the app
- Extend the existing PPM / technician workflow already built
- Reuse the current dark enterprise UI, mobile technician experience, AI-OS branding, and component system
- This feature should feel native to the current product
- This is intended to work as a real, intelligent in-workflow assistant for field engineers during preventive maintenance

==================================================
1. FEATURE PURPOSE
==================================================

On the PPM page, each asset class should have its own expert AI agent that can guide the field engineer by voice and text while working on the asset.

Examples:
- HVAC Expert Agent
- Lift Expert Agent
- Fire Safety Expert Agent
- Plumbing Expert Agent
- Electrical Expert Agent
- Generator / Power Expert Agent
- Chiller Expert Agent

The agent should:
- understand the asset class
- understand the current PPM checklist
- guide the engineer step by step
- answer practical troubleshooting questions
- explain what readings mean
- highlight risk / abnormal observations
- recommend next best action
- help the engineer decide whether to continue, escalate, or raise a corrective incident
- support voice interaction inside the PPM workflow

This should feel like:
“A senior technical expert is on the job with the engineer”

==================================================
2. WHERE THIS APPEARS
==================================================

Integrate this feature into the existing PPM / technician workflow.

Places to add it:
1. PPM task detail page
2. Checklist execution view
3. Asset scan / diagnostics screen
4. Technician mobile flow

Add a visible panel / button:
- “Expert Copilot”
- or “Talk to Asset Expert”

When opened, it should launch a right-side drawer on desktop or bottom sheet on mobile.

3. AGENT ROUTING LOGIC
==================================================

The system must automatically choose the correct expert agent based on:

==================================================
6. AGENT KNOWLEDGE / BEHAVIOR
==================================================

Each asset expert must behave like a specialist field engineer / senior technician.

General behavior rules:
- be concise and operational
- give practical instructions, not theory unless asked
- prioritize safety first
- distinguish between normal vs abnormal conditions
- explain what to check next
- never skip mandatory checklist steps
- if something is unsafe, say so directly
- if uncertainty exists, recommend escalation
- if issue indicates corrective maintenance, recommend raising incident

The agent must use the current job context:
- asset type
- current checklist item
- known readings
- prior issues if available
- parts status if available

Examples of useful answers:
- “That pressure is below expected range for this asset. Recheck the reading, inspect for refrigerant loss, and do not close the PPM as normal.”
- “The coil blockage level suggests cleaning is required before restart.”
- “This observation should trigger a corrective incident, not only a PPM note.”
- “Before moving on, upload the required evidence photo for this step.”

The more context the system has, the better the answers should be.

==================================================
9. PPM-SPECIFIC INTELLIGENCE
==================================================

The agent must be tightly linked to the checklist, not a generic chatbot.

Required capabilities:
1. Step guidance
- explain current checklist step in plain language
- tell engineer what to inspect and why

2. Reading interpretation
- explain pressure, temperature, noise, vibration, etc. where relevant
- say whether value seems normal or abnormal

3. Evidence coaching
- remind user when photos, signatures, or readings are required
- warn if a mandatory item is incomplete

4. Escalation recommendation
- if abnormal condition is detected, recommend:
  - add note
  - raise corrective incident
  - do not close PPM as passed

5. End-of-job summary
- summarize work completed
- highlight abnormalities
- suggest whether asset should remain under observation
