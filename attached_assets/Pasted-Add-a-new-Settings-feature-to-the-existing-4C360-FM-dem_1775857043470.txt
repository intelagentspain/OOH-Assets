Add a new Settings feature to the existing 4C360 FM demo app called:

“Dispatch & AI Rules”

Purpose:
This page allows Admin / Super Admin users to configure how Smart Dispatch and AI Top Match behave.
It should let users define when dispatch is manual, hybrid, or fully automatic, and control which parameters matter for different incident types, severity levels, sites, vendors, and situations.

Important:
- Do NOT rebuild the app
- Reuse the existing dark enterprise UI, settings patterns, forms, cards, and layout
- This is for DEMO use, so mock persistence is acceptable, but the feature must feel real and configurable
- The settings should connect conceptually to the existing AI Smart Dispatch and incident assignment flows already in the app

3. DISPATCH MODES SECTION
--------------------------------------------------

Create a settings card for global dispatch mode logic.

Add controls for:
- Global default mode:
  - Manual
  - Hybrid
  - AI

Add descriptive behavior:
- Manual = all incidents assigned by human
- Hybrid = AI suggests, human confirms
- AI = system auto-assigns according to rules

Add ability to override mode by:
- Site
- Region
- Service line
- Ticket type
- Severity

Example:
- Critical fire incidents = Hybrid only
- Low-priority cleaning tickets = AI
- VIP client tickets = Manual

UI:
- rules table or rule cards
- Add Rule button
- Each rule should support:
  - If [condition]
  - Then mode = Manual / Hybrid / AI

--------------------------------------------------

Create rule-based automation settings for incident assignment.

- Asset type
- Client type
- SLA threshold
- Team / vendor target
- Require confirmation? yes/no

Actions:
- auto assign

6. SEVERITY & TICKET TYPE RULES
--------------------------------------------------

Add a settings section where the admin decides what kinds of tickets can be auto-handled by AI.

This is very important.

Allow rules like:
- Non-critical tickets of type “Cleaning”, “General Maintenance”, “Low-priority HVAC” can be fully AI-assigned
- Medium severity plumbing tickets = Hybrid
- Critical fire / lift / safety incidents = Manual or Hybrid only
- Repeat incidents on same asset always require supervisor review

UI:
- matrix table or rule cards
- columns:
  - Ticket type
  - Severity
  - Allowed mode
  - Escalation requirement
  - Evidence requirement
  - Supervisor approval required

Allow user to define:
- Which SLA levels increase AI urgency
