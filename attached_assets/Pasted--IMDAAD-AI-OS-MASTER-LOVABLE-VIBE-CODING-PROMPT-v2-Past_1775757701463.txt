## PROJECT OVERVIEW

Build a fully interactive, multi-perspective AI-powered Facility Management platform called **"Imdaad AI-OS"**, powered by **4C360**. This is a high-fidelity SaaS product demo — not a wireframe. Every screen must be polished, interactive, and feel like a production enterprise tool.

This is a **single-page React application** with **no backend required** — use mock/hardcoded data throughout. All logic runs client-side.

### Context That Shapes Every Design Decision
This demo is built for a specific pitch to Imdaad, one of the UAE's largest FM companies. Their current workflow has the following real, confirmed pain points — every feature must visibly address at least one:

1. **WhatsApp is their incident system** — staff photograph issues and send to a WhatsApp group. A data engineer then manually reads the group, downloads photos, and creates work orders in Maximo. This takes 10–20 minutes per incident.
2. **Split systems** — one site uses Maximo, all other sites use Oracle. Leadership aggregates via Power BI exports. No unified real-time view exists.
3. **Shared devices, no personal accountability** — devices are zone-based and shared across shifts. Tasks are assigned to "Zone 1 device," not to a named person. No audit trail of who did what.
4. **2,000+ photos on personal phones** — evidence photos go to WhatsApp first, then are manually downloaded and uploaded to Maximo. Photos are lost when staff leave.
5. **No GIS or live asset visibility** — no map view of assets, incidents, or engineer locations exists on any Imdaad site.
6. **PPM is manually assigned monthly** — a data engineer or admin manually assigns planned preventive maintenance at the start of each month. No predictive triggers.
7. **Best pilot opportunity = large residential communities** — Silicon Oasis, Expo City, Furjan. Engineers spread across 5km², managing hundreds of villas. This is where GIS and mobile-first operations matter most.

## ANIMATIONS & MICRO-INTERACTIONS

- **Map engineer markers**: continuous ripple/pulse animation (scale + opacity keyframes, 2s loop)
- **Critical incident markers**: urgent red pulse, faster frequency
- **Live badge on map**: green dot blinking (1.5s loop)
- **Smart Scan button**: continuous blue glow pulse (2s loop, scale 1→1.08)
- **Perspective switch**: Framer Motion `AnimatePresence` — fade + slight Y translate (200ms)
- **PPM countdown chips**: subtle color shift as days decrease
- **SLA countdown < 5 min**: `@keyframes shake` on the task card
- **Checklist check**: scale bounce (0.8 → 1.1 → 1.0) + green fill transition
- **Assign button click**: ripple effect
- **Card hover**: `translateY(-2px)` + border color intensifies
- **Login PIN entry**: each digit shows a filled circle; shake animation on wrong PIN
- **Technician marker on client map**: animates position every 3 seconds (simulated movement)
