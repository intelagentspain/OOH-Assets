Add a real production-ready feature to the existing 4C360 FM app called:

“QR AI Issue Capture”

This is NOT a demo-only mocked feature.
Build it as a real end-to-end workflow where a user scans a QR code, takes a real photo, the backend sends the image to a real vision-capable AI service, the AI analyzes the image, and the system creates a real structured incident.

Do not rebuild the app.
Extend the existing architecture cleanly.
Reuse current styling, navigation, and data structures where possible.

--------------------------------------------------
1. GOAL
--------------------------------------------------

Build a real mobile-first issue reporting flow for on-site users such as:
- cleaner
- tenant
- security
- technician
- visitor / public user

Flow:
1. User scans QR code
2. Lands on public issue reporting page
3. Completes short onboarding
4. Opens camera / uploads image
5. Image is sent to backend
6. Backend sends image to a real AI vision service
7. AI returns:
   - issue classification
   - category
   - severity
   - structured description
   - anomaly observations
   - recommended next action
8. User reviews result
9. User submits incident
10. Incident is saved in the system and appears in incidents, pulse feed, dispatch, and dashboard

Create a public route:

This route must work without login.

It should load context from siteId / assetId:
- site name
- asset name if available
- area / location name

5. STEP 3 — BACKEND IMAGE ANALYSIS
--------------------------------------------------

Build a backend endpoint to analyze the uploaded image with a real vision AI service.

Create backend endpoint:
POST /api/ai/analyze-issue-image

Input:
- image file
- siteId
- assetId
- optional reporter metadata

Backend must:
1. validate file type and size
2. upload/store image temporarily or process directly
3. send the image to a real vision-capable AI model/service
4. request structured JSON output

Expected AI output format:
{
  "title": "string",
  "description": "string",
  "issueType": "string",
  "category": "string",
  "severity": "low | medium | high | critical",
  "identifiedAsset": "string",
  "observations": ["string", "string"],
  "recommendedAction": "string",
  "confidence": number
}

The model should classify issues relevant to FM, such as:
- plumbing leak
- HVAC issue
- cleanliness / housekeeping issue
- broken tile / visible damage
- fire safety issue
- electrical issue
- general maintenance

The prompt sent to the AI must instruct it to:
- describe what is actually visible
- avoid making unsafe claims without evidence
- identify likely facility issue category
- detect unusual or risky observations
- recommend a practical action
- return structured JSON only

--------------------------------------------------
6. AI PROMPTING REQUIREMENT
--------------------------------------------------

Implement server-side structured prompting for FM issue classification.

The AI prompt should tell the model:
- You are an FM incident triage assistant
- Analyze the uploaded photo
- Identify likely issue type
- Write a concise but professional operational description
- Identify abnormal conditions or safety concerns
- Suggest severity based on visible evidence only
- Suggest a recommended action
- Return valid JSON only

The response must be parsed server-side and validated before returning to frontend.

If parsing fails:
- return a safe fallback error
- do not crash the flow
